

## Карточка сотрудника в кабинете клиента

### Что будет сделано

Клиент, переходя в личный кабинет по ссылке, будет видеть карточку закрепленного за ним сотрудника с фото и описанием. Сотрудники смогут управлять своим фото и описанием на вкладке "Кабинеты".

### Технические шаги

**1. Миграция базы данных**

Добавить в таблицу `profiles` два новых столбца:
- `avatar_url` (text, nullable) -- URL фото сотрудника
- `bio` (text, nullable) -- описание/должность сотрудника

**2. Создание бакета для аватаров**

Создать публичный storage bucket `avatars` с RLS-политиками, позволяющими авторизованным пользователям загружать и удалять свои файлы, а всем -- читать.

**3. Вкладка "Кабинеты" -- редактирование профиля сотрудника**

В компоненте `ClientCabinetsManagement.tsx` добавить блок вверху страницы "Мой профиль для клиентов":
- Загрузка/смена фото (аватар) через storage bucket `avatars`
- Текстовое поле для описания/должности
- Кнопка "Сохранить" -- обновляет `profiles.avatar_url` и `profiles.bio`

**4. Edge Function `get-client-cabinet` -- возвращать данные сотрудника**

Обновить функцию: после получения клиента, по `employee_id` запросить из `profiles` поля `full_name`, `avatar_url`, `bio` и вернуть их в ответе как объект `employee`.

**5. Страница кабинета клиента (`ClientCabinet.tsx`)**

Добавить карточку "Ваш специалист" между хедером и этапами:
- Аватар сотрудника (компонент `Avatar` с fallback-инициалами)
- Имя сотрудника
- Описание/должность

### Итог

- Сотрудник заполняет фото и описание один раз на вкладке "Кабинеты"
- Все клиенты этого сотрудника видят его карточку в своих кабинетах
