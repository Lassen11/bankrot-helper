
# Кабинет Клиента — план реализации

## Что будет сделано

Создание публичной страницы "Кабинет клиента", доступной по уникальной ссылке без авторизации. В кабинете клиент видит свой прогресс по процедуре банкротства, разбитый на 12 этапов. Сотрудник может редактировать названия этапов и отмечать их выполнение.

## Как это будет работать

1. **Уникальная ссылка** — для каждого клиента генерируется секретный токен (UUID). Ссылка вида `/cabinet/abc123-...` открывает кабинет без логина.
2. **12 этапов банкротства** — каждый этап имеет номер, название, описание и статус выполнения. Этапы создаются автоматически при генерации ссылки с предустановленными названиями.
3. **Сотрудник** может редактировать названия/описания этапов и отмечать их как выполненные (из страницы клиента `/client/:id`).
4. **Клиент** видит только прогресс (без возможности редактирования).

## Этапы реализации

### 1. База данных — две новые таблицы

**Таблица `client_cabinet_tokens`** — хранит токены доступа:
- `id`, `client_id`, `token` (уникальный UUID), `created_at`, `is_active`

**Таблица `bankruptcy_stages`** — 12 этапов для каждого клиента:
- `id`, `client_id`, `stage_number` (1-12), `title`, `description`, `is_completed`, `completed_at`, `created_at`, `updated_at`

RLS: таблицы доступны сотрудникам (по `employee_id` клиента) и админам. Для публичного доступа по токену будет использоваться Edge Function.

### 2. Edge Function `get-client-cabinet`

Принимает токен, проверяет его валидность, возвращает данные клиента и этапы. Это позволяет обойти RLS для неавторизованных пользователей.

### 3. Новая страница `/cabinet/:token` — Кабинет Клиента

Публичная страница (без ProtectedRoute):
- Имя клиента, прогресс-бар общего выполнения
- 12 этапов в виде вертикального таймлайна с иконками статуса
- Выполненные этапы отмечены зеленым, текущий — выделен, будущие — серые

### 4. Генерация ссылки и управление этапами в `/client/:id`

В существующую страницу клиента добавляется:
- Кнопка "Сгенерировать ссылку на кабинет" — создает токен и 12 этапов
- Копирование ссылки в буфер обмена
- Блок управления этапами: редактирование названий, отметка выполнения

### 5. Предустановленные 12 этапов банкротства

1. Сбор документов
2. Анализ финансового состояния
3. Подготовка заявления в суд
4. Подача заявления в Арбитражный суд
5. Назначение финансового управляющего
6. Первое судебное заседание
7. Реструктуризация долгов
8. Реализация имущества
9. Работа с кредиторами
10. Формирование реестра кредиторов
11. Завершение процедуры реализации
12. Списание долгов и закрытие дела
