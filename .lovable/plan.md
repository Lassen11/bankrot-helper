
## Несколько сотрудников на одного клиента

### Что будет сделано

Сейчас у клиента один `employee_id` -- один сотрудник. Нужна связь "многие ко многим", чтобы к делу клиента можно было прикреплять нескольких специалистов, и клиент видел карточки всех в своем кабинете.

### Технические шаги

**1. Новая таблица `client_employees`**

Создать таблицу-связку:
- `id` (uuid, PK)
- `client_id` (uuid, NOT NULL) -- ссылка на клиента
- `employee_id` (uuid, NOT NULL) -- ссылка на сотрудника (user_id)
- `role_label` (text, nullable) -- подпись роли (например "Юрист", "Финансовый управляющий")
- `created_at` (timestamptz)
- Уникальный индекс на `(client_id, employee_id)`

RLS-политики: доступ для owner-сотрудника клиента (`clients.employee_id`) и админов.

**2. Управление сотрудниками в карточке кабинета**

В раскрытой карточке клиента на вкладке "Кабинеты" добавить новую вкладку "Команда" (или секцию):
- Список прикрепленных сотрудников с аватаром, именем и ролью
- Выпадающий список для добавления нового сотрудника (из всех пользователей системы)
- Поле "роль" при добавлении
- Кнопка удаления сотрудника из дела

**3. Edge Function `get-client-cabinet` -- возвращать массив сотрудников**

Вместо одного объекта `employee` возвращать массив `employees`:
- Запрос из `client_employees` по `client_id`
- Для каждого `employee_id` подтянуть из `profiles`: `full_name`, `avatar_url`, `bio`
- Добавить `role_label` из таблицы-связки
- Для обратной совместимости: также включить основного сотрудника из `clients.employee_id`

**4. Страница кабинета клиента (`ClientCabinet.tsx`)**

Вместо одной карточки "Ваш специалист" -- блок "Ваши специалисты":
- Отображать все карточки сотрудников в ряд/список
- Каждая карточка: аватар, имя, роль/должность (role_label или bio)

### Изменяемые файлы

- Новая миграция SQL: таблица `client_employees` + RLS
- `src/components/ClientCabinetsManagement.tsx` -- вкладка/секция "Команда" с управлением
- `supabase/functions/get-client-cabinet/index.ts` -- массив `employees`
- `src/pages/ClientCabinet.tsx` -- список карточек специалистов
