

## Добавление блоков "Прогресс платежей", "Детали договора" и "График платежей" в Кабинет Клиента

### Что будет сделано

Клиент, переходя в личный кабинет по ссылке, будет видеть три новых блока с финансовой информацией по своему делу:
- **Прогресс платежей** -- депозит, общий прогресс оплаты, остаток
- **Детали договора** -- сумма договора, дата, период рассрочки, ежемесячный платеж
- **График платежей** -- список всех платежей с датами и статусами (только для просмотра, без возможности редактирования)

### Технические шаги

**1. Edge Function `get-client-cabinet` -- расширить возвращаемые данные**

Добавить в запрос клиента дополнительные поля:
- `contract_amount`, `total_paid`, `deposit_paid`, `deposit_target`, `monthly_payment`, `installment_period`, `first_payment`, `remaining_amount`

Добавить запрос платежей из таблицы `payments` по `client_id`:
- Поля: `payment_number`, `original_amount`, `custom_amount`, `due_date`, `is_completed`, `payment_type`, `completed_at`
- Сортировка по `payment_number`

Вернуть расширенный объект `client` и массив `payments` в ответе.

**2. Страница `ClientCabinet.tsx` -- добавить три новых блока**

Между блоком специалистов и этапами добавить:

- **Карточка "Прогресс платежей"** -- использовать существующий компонент `PaymentProgress` с данными из API (totalPaid, contractAmount, depositPaid, depositTarget)

- **Карточка "Детали договора"** -- отображение: сумма договора, дата договора, период рассрочки, ежемесячный платеж, авансовый платеж, остаток к оплате

- **Карточка "График платежей"** -- read-only таблица/список платежей: номер, тип (авансовый/ежемесячный), дата, сумма, статус (оплачен/ожидает). Без кнопок редактирования -- только просмотр

Обновить интерфейс `CabinetData` для включения новых полей.

### Изменяемые файлы

- `supabase/functions/get-client-cabinet/index.ts` -- расширить select клиента и добавить запрос payments
- `src/pages/ClientCabinet.tsx` -- добавить три карточки с финансовой информацией
